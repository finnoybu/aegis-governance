## Summary

ATX-1 v2.1 adds a novel tactic (TA010), a new structural root cause (RC5), and 4 techniques discovered during 5 rounds of white-box adversarial testing against the AEGIS Claude Code Plugin (RFC-0006) on 2026-03-26. This is the first tactic class in ATX-1 not derived from the Agents of Chaos study — it emerged from AEGIS's own governance enforcement research.

---

### Added

- **RC5: No Environment Model** — The governance layer operates on an abstraction of the execution environment but lacks a complete model of the environment's actual capabilities. Actions evaluated as one type may produce entirely different effects depending on how the environment interprets them.

- **TA010: Exploit Governance Visibility Gap** — The agent performs actions whose actual effects are invisible to the governance layer because the execution environment provides capabilities below the governance model's abstraction level.

- **T10001: Exploit Pseudo-Filesystem Capabilities** (High) — OS pseudo-filesystem paths (/dev/tcp, /dev/udp, /proc) provide network or system interfaces through file path syntax invisible to governance.

- **T10002: Establish Persistence via Environment Auto-Execution** (High) — Writes to files the environment auto-executes: shell profiles, git hooks, package scripts, IDE tasks, Dockerfiles, tool configs.

- **T10003: Inject Persistent Agent Instructions** (Critical) — Modification of agent instruction files that auto-load at session start, persist across sessions, and propagate via version control. Generic description per coordinated disclosure RD-2603-01.

- **T10004: Exploit Governance-Runtime Parser Divergence** (High) — Inputs parsed differently by governance evaluator vs. execution runtime: newline/CRLF injection, null bytes, Unicode homoglyphs, heredoc syntax, shell metacharacters.
